Keyboard-navigable splitters with pointer capture, constraint clamping, and persistent layout state in localStorage. Arrow keys + Shift for granular control.

7 purpose-built SVG icons (run, pause, step, reset, import, export, close). No icon library dependency — every glyph has a reason to exist.

// PRIORITY ISSUES
P2

No distinctive visual signature beyond "dark + green"

The app looks like a competent developer tool but has no visual identity beyond its genre. Run the same layout with a cyan accent and it looks like a different product. The palette doesn't encode anything specific to 6809 emulation.

FIX: Consider an era-appropriate dual-accent system — green for execution state, amber (VT100) for diagnostics and memory. The existing accent-amber (#ffb000) is currently underused.
P2

Uniform panel dressing flattens hierarchy

Every panel uses the identical .panel > .panel-header + .panel-body structure with the same radius, border, and header treatment. Consistent but flat — nothing signals which panel is primary (disassembly) vs secondary (breakpoints).

FIX: Introduce subtle visual hierarchy — a thin top-border detail on primary panels, or a distinct header color for the active debug panel.
